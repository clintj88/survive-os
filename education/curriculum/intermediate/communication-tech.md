# Intermediate Band (Grades 6-8, Ages 11-14) — Communication Technology

## Overview
Communication technology enables communities to coordinate, trade, call for help, and share knowledge. Students progress from radio fundamentals through hands-on device operation to network architecture and basic programming. These skills directly support the SURVIVE OS communication modules.

## Grade 6: Radio Fundamentals and Electronics

### Unit 1: Radio Basics (Weeks 1-8)
**Duration**: 8 weeks
**Materials**: AM/FM radio receiver, diagrams of electromagnetic spectrum, wire for antenna experiments, battery, LED
**Survival Relevance**: Radio is the primary long-distance communication tool when infrastructure fails. Understanding radio fundamentals enables operation, troubleshooting, and improvisation.

#### Learning Objectives
- Explain the electromagnetic spectrum and where radio waves fit
- Understand AM and FM modulation concepts
- Describe how a radio transmitter and receiver work at a basic level
- Build a simple crystal radio or antenna

#### Teacher Notes
Connect to physics (wave properties) and history (communication revolutions). Emphasize: radio works without infrastructure — no cell towers, no internet, no power grid needed (with batteries or solar).

#### Lessons
**Lesson 1-2: The Electromagnetic Spectrum**
- Activity: The spectrum from radio waves (long, low energy) through microwaves, infrared, visible light, ultraviolet, X-rays, to gamma rays (short, high energy). Radio waves are just light we can't see — they travel at the speed of light and can carry information.
- Key Concepts: All electromagnetic waves travel at light speed, wavelength and frequency are inversely related (longer wave = lower frequency), different frequencies behave differently (some bounce off the ionosphere, some pass through walls)
- Practice: Diagram the spectrum, label each type, and identify survival-relevant applications

**Lesson 3-4: AM and FM**
- Activity: AM (Amplitude Modulation) — vary the wave's strength to encode sound. FM (Frequency Modulation) — vary the wave's frequency to encode sound. AM travels farther but is noisier. FM sounds better but has shorter range.
- Key Concepts: Modulation is how information is put onto a radio wave, AM for long range, FM for quality, both require a transmitter and receiver tuned to the same frequency
- Practice: Listen to AM and FM stations (if available) and compare quality and range

**Lesson 5-6: How Radios Work**
- Activity: Transmitter: microphone → amplifier → modulator → antenna (converts electrical signal to radio wave). Receiver: antenna (captures radio wave) → tuner (selects frequency) → demodulator → amplifier → speaker.
- Key Concepts: Antennas convert between electrical signals and radio waves, tuning selects the desired signal from all signals, amplification makes weak signals usable
- Practice: Trace the signal path through a transmitter and receiver diagram

**Lesson 7-8: Build a Simple Radio**
- Activity: Build a crystal radio (diode, coil, earphone, antenna wire — no batteries needed) or a simple wire antenna. "This is the simplest possible radio receiver — it uses the radio wave's own energy to produce sound."
- Key Concepts: Radio reception requires only an antenna, a tuned circuit, and a detector, the signal itself provides the energy, this demonstrates the fundamental principles
- Practice: Build and test a simple receiving device

#### Assessment
- Describe the electromagnetic spectrum and radio wave properties
- Explain AM and FM modulation
- Trace signal path through a transmitter and receiver
- Build a functional receiving device

### Unit 2: Morse Code (Weeks 9-14)
**Duration**: 6 weeks
**Materials**: Buzzers or flashlights for practice, Morse code charts, paper
**Survival Relevance**: Morse code is the most resilient communication protocol — it works with any signaling method (light, sound, radio, tapping) and can be received under conditions where voice fails

#### Learning Objectives
- Encode and decode Morse code for the full alphabet and numbers
- Send and receive at 5+ words per minute
- Use Morse code via multiple signaling methods (sound, light, written)
- Understand the role of Morse code in emergency communication

#### Lessons
**Lesson 1-2: Learning the Code**
- Activity: Start with the most common letters (E, T, A, O, I, N, S, H, R). E = dit, T = dah. Build up. Use mnemonics if helpful. Practice sending and receiving 5 letters at a time.
- Key Concepts: Morse code is binary (short and long), letter frequency determines learning order, practice is the only path to proficiency
- Practice: Send and receive 10 letters per session, building to full alphabet

**Lesson 3-4: Words and Phrases**
- Activity: Send full words. Prosigns: SOS (··· --- ···), CQ (calling any station), 73 (best regards). Practice standard phrases used in radio communication.
- Key Concepts: Word spacing separates words, prosigns are standard abbreviations, standardized phrases reduce errors
- Practice: Send and receive 10 common words and phrases

**Lesson 5-6: Multi-Method Communication**
- Activity: Send Morse by buzzer (sound), flashlight (light), tapping (vibration), and written (dots and dashes). Each method works in different conditions — sound works in fog, light works across open distance at night, tapping works through walls.
- Key Concepts: The same code works with ANY signaling method, adaptability is the key advantage, Morse code has been in use since 1844 and remains relevant
- Practice: Communicate a 20-word message using at least 2 different methods

#### Assessment
- Encode and decode the full alphabet and numbers in Morse code
- Send and receive at 5+ words per minute
- Communicate a complete message using Morse code via 2 different methods

### Unit 3: Signal Systems and Basic Electronics (Weeks 15-24)
**Duration**: 10 weeks
**Materials**: Wire, resistors, LEDs, batteries, switches, capacitors, transistors (if available), soldering iron and solder (if available), breadboard
**Survival Relevance**: Understanding basic electronics enables repair and modification of communication equipment, sensors, and other electronic devices the community depends on

#### Learning Objectives
- Design and use signal systems (flag, light, sound) for community communication
- Understand basic electronic components (resistor, capacitor, LED, transistor, diode)
- Build simple electronic circuits (LED flasher, amplifier, voltage divider)
- Solder electronic connections (if equipment available)

#### Lessons
**Lesson 1-2: Signal Systems**
- Activity: Design a community communication system using flags, lights, or sounds. Fixed signals (flag positions = specific messages), sequential signals (light flashes = Morse), alarm signals (horn patterns = specific warnings). Document and train the community.
- Key Concepts: Standardized signals prevent confusion, simplicity enables reliability, redundancy (multiple methods) ensures communication
- Practice: Design, document, and test a community signal system

**Lesson 3-4: Electronic Components**
- Activity: Identify and test components: Resistor (limits current — color bands indicate value). Capacitor (stores charge — like a small battery). LED (emits light when current flows). Diode (allows current in one direction only). Transistor (switch or amplifier controlled by a small signal).
- Key Concepts: Each component has a specific function, understanding components enables circuit reading and troubleshooting
- Practice: Identify 20 components and explain their function

**Lesson 5-6: Circuit Building**
- Activity: Build on a breadboard: LED circuit with current-limiting resistor (calculate: R = (V_supply - V_LED) / I_LED). Voltage divider (two resistors that produce a lower voltage). RC timer circuit (resistor + capacitor creates time delay).
- Key Concepts: Circuits are combinations of components performing a function, calculation determines correct component values, breadboards enable testing before permanent assembly
- Practice: Build 3 functional circuits and verify operation

**Lesson 7-8: Soldering**
- Activity: If soldering equipment is available: learn technique — heat the joint (not the solder), apply solder to the heated joint, let flow, remove heat. Practice on scrap components. A good joint is shiny and smooth. A bad joint is dull and lumpy.
- Key Concepts: Soldering creates permanent electrical connections, proper technique prevents cold joints (which fail), safety — avoid burns, ventilate
- Practice: Solder 10 practice joints and 1 functional circuit

**Lesson 9-10: Practical Electronics Project**
- Activity: Build something useful — a battery tester, LED flashlight, simple alarm, or signal device. Design the circuit, select components, build on breadboard, test, then solder permanently.
- Key Concepts: Electronics projects follow design → prototype → test → build, documentation matters for future repair
- Practice: Complete a functional electronics project

#### Assessment
- Design a community signal system
- Identify electronic components and explain their functions
- Build functional circuits on a breadboard
- Complete a practical electronics project

## Grade 7: Ham Radio and Meshtastic

### Unit 4: Ham Radio Operation (Weeks 1-10)
**Duration**: 10 weeks
**Materials**: Ham radio transceiver (even low-power HT), antenna, power supply, study guide for license examination
**Survival Relevance**: Ham radio provides long-range voice communication without any infrastructure. It is the primary inter-community communication tool.

#### Learning Objectives
- Understand ham radio frequency bands and their characteristics
- Operate a radio transceiver (transmit and receive)
- Follow radio procedure and etiquette (callsigns, phonetic alphabet, procedure words)
- Study for ham radio license examination (regulations may apply in normal times)

#### Lessons
**Lesson 1-2: Frequency Bands**
- Activity: Different frequency bands have different characteristics. HF (3-30 MHz): long range, can bounce off ionosphere for global communication. VHF (30-300 MHz): line of sight, good for local communication. UHF (300-3000 MHz): line of sight, penetrates buildings better. Each has advantages.
- Key Concepts: Frequency determines range and behavior, band selection depends on communication need, atmospheric conditions affect propagation
- Practice: Identify appropriate bands for 5 different communication scenarios

**Lesson 3-4: Radio Operation**
- Activity: Hands-on with a transceiver: power on, select frequency, adjust volume and squelch, transmit (press PTT, speak clearly, release PTT). Listen before transmitting. Identify your station.
- Key Concepts: Listen before transmitting, keep transmissions brief and clear, always identify your station, release PTT to listen for response
- Practice: Make 10 practice transmissions with correct procedure

**Lesson 5-6: Radio Procedure**
- Activity: NATO phonetic alphabet (Alpha, Bravo, Charlie...). Procedure words: "Over" (my turn is done, your turn), "Out" (conversation finished), "Roger" (understood), "Say again" (repeat), "Break break" (emergency interruption). Standard call format: "(their callsign), this is (your callsign), (message), over."
- Key Concepts: Standard procedure prevents confusion, phonetic alphabet prevents letter confusion, emergency traffic has priority
- Practice: Conduct 5 simulated radio conversations with full procedure

**Lesson 7-8: Emergency Communication**
- Activity: Emergency protocols: "MAYDAY MAYDAY MAYDAY" (life-threatening emergency), "PAN PAN PAN" (urgent but not life-threatening). Practice: send a distress call with location, nature of emergency, resources needed, number of persons involved.
- Key Concepts: Emergency communication must be clear, concise, and standardized, everyone in the community should know how to make a distress call
- Practice: Send 3 emergency communications with complete information

**Lesson 9-10: License Study**
- Activity: Study the Technician-class ham radio license material: regulations, frequency allocations, safety, basic electronics, antenna theory. Even post-collapse, this knowledge is practical.
- Key Concepts: License knowledge covers the fundamentals of radio operation, regulations promote efficient spectrum use, the knowledge is valuable regardless of licensing status
- Practice: Complete practice examinations

#### Assessment
- Operate a radio transceiver correctly
- Conduct communications using proper procedure
- Send emergency communications with complete information
- Pass a practice license examination

### Unit 5: Meshtastic Device Operation (Weeks 11-18)
**Duration**: 8 weeks
**Materials**: Meshtastic devices (LoRa radios), computers for configuration (if available), antenna supplies
**Survival Relevance**: Meshtastic provides text-based mesh networking — each device relays messages, creating a self-healing network without any central infrastructure. This is a key SURVIVE OS communication system.

#### Learning Objectives
- Understand mesh networking concepts (nodes, relays, hops, mesh topology)
- Configure and operate Meshtastic devices
- Plan and deploy a community Meshtastic network
- Troubleshoot common network issues

#### Lessons
**Lesson 1-2: Mesh Networking**
- Activity: Arrange students in a grid. Pass a written message from one corner to the opposite corner — each student can only pass to adjacent students. Even if some students leave, the message finds a path. "This is mesh networking — every node is a relay."
- Key Concepts: Mesh networks have no single point of failure, each node extends the network, messages route around failures automatically
- Practice: Diagram a 10-node mesh network and trace message paths under different failure scenarios

**Lesson 3-4: Meshtastic Basics**
- Activity: Power on a Meshtastic device. Send a text message. Receive a text message. Check signal strength. Observe message routing (how many hops). Set device name and channel.
- Key Concepts: Meshtastic uses LoRa (Long Range) radio, range is typically 1-10 km per hop, text only (no voice), low power consumption (days on a battery)
- Practice: Send and receive messages between devices at various distances

**Lesson 5-6: Network Planning**
- Activity: Design a community Meshtastic network: where to place relay nodes for maximum coverage (high points), how many nodes needed, power supply for each node (solar + battery), channel configuration for different use groups (general, medical, security).
- Key Concepts: Node placement determines coverage, solar power enables unattended operation, channel separation organizes traffic
- Practice: Create a network plan with node locations, power solutions, and channel assignments

**Lesson 7-8: Deployment and Troubleshooting**
- Activity: Deploy nodes according to the plan. Test coverage by walking the community and sending messages from various locations. Identify dead zones. Adjust node placement. Document final network performance.
- Key Concepts: Plans require field validation, real-world performance differs from predictions, documentation enables future troubleshooting
- Practice: Deploy, test, and document a functional network segment

#### Assessment
- Explain mesh networking concepts
- Configure and operate Meshtastic devices
- Design and deploy a network segment with documented coverage

### Unit 6: Antenna Theory and Radio Etiquette (Weeks 19-26)
**Duration**: 8 weeks
**Materials**: Wire, coax cable (if available), antenna modeling materials, field strength meter or radio with signal strength indicator
**Survival Relevance**: Antenna performance determines communication range more than any other single factor. A good homemade antenna can outperform a bad commercial one.

#### Learning Objectives
- Understand antenna fundamentals (wavelength, resonance, polarization, directional vs omnidirectional)
- Build simple antennas (dipole, ground plane, Yagi)
- Compare antenna performance through testing
- Practice advanced radio etiquette and net control procedures

#### Lessons
**Lesson 1-2: Antenna Fundamentals**
- Activity: An antenna converts electrical signal to radio wave (and vice versa). Its length must match the radio wavelength — a half-wave dipole for a 2-meter frequency is about 1 meter long. Calculate antenna lengths for different frequencies.
- Key Concepts: Antenna length relates to wavelength, a matched antenna is efficient, a mismatched antenna wastes power, height improves range
- Practice: Calculate correct antenna lengths for 5 different frequencies

**Lesson 3-4: Building Antennas**
- Activity: Build a half-wave dipole from wire — cut to calculated length, connect center to coax cable, hang from a tree or pole. Test reception before and after — improvement should be dramatic.
- Key Concepts: Homemade antennas work well when properly sized, height above ground improves performance, coax cable connects antenna to radio
- Practice: Build and test a dipole antenna, measure performance improvement

**Lesson 5-6: Directional Antennas**
- Activity: Build a simple Yagi antenna (directional — like a TV antenna). Point it toward the target station. Compare signal strength with omnidirectional antenna. "Directional antennas concentrate signal in one direction — like a flashlight vs a bare bulb."
- Key Concepts: Directional antennas trade coverage area for range, they must be aimed, they are useful for point-to-point links
- Practice: Build a Yagi antenna and compare performance to the dipole

**Lesson 7-8: Net Control and Advanced Procedures**
- Activity: Practice running a radio net: Net Control Station (NCS) calls the net to order, takes check-ins, manages traffic, handles emergency breaks, closes the net. Each student practices as NCS and as a participant.
- Key Concepts: Nets organize group communication, NCS maintains order, scheduled nets enable routine coordination, emergency nets enable crisis response
- Practice: Run a complete radio net with 5+ participants

#### Assessment
- Calculate antenna dimensions for a given frequency
- Build and test two antenna types
- Run a radio net as Net Control Station

## Grade 8: Networking, Computing, and Programming

### Unit 7: Network Fundamentals (Weeks 1-8)
**Duration**: 8 weeks
**Materials**: Computers (if available), network cables, router/switch (if available), diagrams
**Survival Relevance**: The SURVIVE OS platform runs on a local network. Understanding networking enables maintenance, troubleshooting, and expansion of the community's digital infrastructure.

#### Learning Objectives
- Understand IP addressing (IPv4 — what the numbers mean)
- Explain routing (how data finds its destination)
- Describe the client-server model (how SURVIVE OS modules communicate)
- Diagram the community's network infrastructure

#### Lessons
**Lesson 1-2: IP Addressing**
- Activity: Every device on a network has an address — like a house address. IPv4: four numbers separated by dots (192.168.1.1). Subnet masks define the neighborhood. "192.168.1.x means all devices in our local network."
- Key Concepts: IP addresses identify devices, subnet masks define network boundaries, private addresses (192.168.x.x, 10.x.x.x) are for local networks
- Practice: Assign IP addresses to 10 devices and identify their subnet

**Lesson 3-4: How Data Travels**
- Activity: "When you request a page from the SURVIVE OS medical module, your computer sends a packet to 192.168.1.40. The router looks at the address and sends it to the right place. The medical server responds with the data."
- Key Concepts: Data travels in packets, routers direct packets based on addresses, switches connect devices on the same network
- Practice: Trace a data path from a user request to a server response

**Lesson 5-6: SURVIVE OS Architecture**
- Activity: Map the SURVIVE OS module network: platform shell (8000), identity (8001), communication modules (8010-8012), security (8020), agriculture (8030), medical (8040), etc. Each module is a service on a specific port.
- Key Concepts: Port numbers identify services, the platform shell ties modules together, understanding the architecture enables troubleshooting
- Practice: Diagram the complete SURVIVE OS network with IP addresses and ports

**Lesson 7-8: Troubleshooting**
- Activity: Common network problems: device can't connect (check IP address, cable, switch), service unavailable (check if the service is running, correct port), slow performance (check for congestion, faulty cables). Systematic approach: start at the physical layer and work up.
- Key Concepts: Systematic troubleshooting is faster than random guessing, most problems are simple (cable unplugged, wrong address, service not started)
- Practice: Diagnose and resolve 5 simulated network problems

#### Assessment
- Explain IP addressing and subnetting
- Diagram the community network infrastructure
- Troubleshoot 3 network problems systematically

### Unit 8: Computer Maintenance and Programming Introduction (Weeks 9-20)
**Duration**: 12 weeks
**Materials**: Computers (SURVIVE OS systems), screwdrivers, compressed air, programming environment (Python if available)
**Survival Relevance**: Maintaining the community's computers and understanding basic programming extends the life of critical equipment and enables customization of SURVIVE OS tools

#### Lessons
**Lesson 1-2: Hardware Maintenance**
- Activity: Open a computer (with permission). Identify components: CPU, RAM, storage, power supply, motherboard, ports. Clean dust with compressed air or careful brushing. Check connections. "Dust is the #1 killer of computers."
- Key Concepts: Heat kills electronics, dust blocks cooling, cleaning extends life, understanding components enables repair
- Practice: Clean and inspect a computer, identify all components

**Lesson 3-4: Software Maintenance**
- Activity: Using SURVIVE OS: check disk space, review system logs, update software (if updates available), verify service status. "systemctl status survive-medical" shows if the medical module is running.
- Key Concepts: Software maintenance prevents crashes, logs reveal problems, disk space must be monitored, services must be running for modules to work
- Practice: Perform a complete software maintenance check on a SURVIVE OS system

**Lesson 5-6: Programming Introduction — Python**
- Activity: If computers are available: print("Hello, community!"). Variables, types (string, integer, float). Input from user. If/else decisions. Simple loops. "Programming is giving very precise instructions to a machine."
- Key Concepts: Programming is structured problem-solving, computers do exactly what you tell them (no more, no less), small programs can be very useful
- Practice: Write 5 simple programs (calculator, temperature converter, resource counter, quiz, timer)

**Lesson 7-8: Data Management**
- Activity: Understand databases at a conceptual level — tables of organized information. Practice with spreadsheets or simple database queries. "The medical module stores patient records in a database. The agriculture module stores crop data."
- Key Concepts: Databases organize information for retrieval, good data structures enable good decisions, data integrity matters
- Practice: Design a simple database for a community need (inventory, health records, weather data)

**Lesson 9-10: Practical Programming Project**
- Activity: Write a useful program: inventory tracker, weather data logger, crop yield calculator, trade balance calculator, or duty roster generator. Keep it simple and functional.
- Key Concepts: Programs should solve real problems, simple programs that work are better than complex ones that don't, test thoroughly
- Practice: Complete a functional program and demonstrate to class

**Lesson 11-12: Record Keeping Systems**
- Activity: Design a comprehensive record-keeping system for the community: what data to collect, how to organize it, who enters it, who accesses it, how to back it up, how to ensure accuracy.
- Key Concepts: Record keeping is the memory of the community, good systems are simple and consistent, backup prevents data loss
- Practice: Design and document a record-keeping system

#### Assessment
- Perform hardware and software maintenance on a computer
- Write a functional Python program that solves a real problem
- Design a record-keeping system for a community need
